## TEASER SEQUENCE STRATEGY (2-Phase)

### Phase 1 — 3 Days Before Launch
Use Option C or a version of Option D. Short, mysterious, just enough to prime the algorithm that your account is active.

```
Something's coming.

For the federal data scientists doing serious work in environments
that make serious work very hard.

Friday. Free. Open source.
```
**Character count: ~153**

### Phase 2 — Evening Before Launch (Primary Teaser)
Use the Primary Teaser or Option B above.

```

Python, ML, AI workflows — all adapted for the real constraints of .gov environments.
Tomorrow morning. Free.
```

**Reply Option B:**
```
The short version: think of every data science course that assumed you have cloud access,
admin rights, and a modern toolchain.

Now imagine the opposite of that.

That's the environment this handbook is built for.
```

**Reply Option C (for defense/military audience specifically):**
```

```

## TIMING NOTES

- Post primary teaser: **7–9 PM EST the night before launch**
- Do not include the link in teasers — let curiosity work
- The teaser tweet should get bookmarks and replies asking "what is it?" — that's a good sign
- If teaser gets strong early engagement, consider Quote Tweeting it at noon the next day just before the full thread drops
