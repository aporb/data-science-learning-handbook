## Exercise 1: The Null Pattern Hunt

You've been handed a 50,000-row Department of the Navy maintenance dataset. A quick `df.isnull().sum()` shows that `completion_date` has approximately 4,000 null values — about 8% of records. Your team lead says "that's within tolerance, proceed with analysis."

**Do not proceed yet.**

**Tasks:**

1. Compute the overall null rate for `completion_date` in the synthetic dataset generated by `generate_synthetic_don_dataset()`.

2. Break down the null rate by `source_system`. Are the nulls randomly distributed, or do they cluster in one or more source systems?

3. Write a one-paragraph finding statement that a non-technical program manager could understand, explaining:
   - What the overall null rate is
   - Whether it's random or clustered
   - What action you recommend before proceeding with analysis

4. If you found that the nulls clustered in one source system, what would you do differently in your analysis compared to if the nulls were random?

**Hints:** Use `analyze_null_patterns()` from `01_statistical_profiling.py`. The null pattern in the synthetic dataset is not random — find it.

## Exercise 2: The Impossible Date Problem

You are building a predictive maintenance model that will use `days_elapsed` (the number of days between a work order opening and its completion) as a feature.

**Tasks:**

1. Load the synthetic DoN dataset. Identify all records where `days_elapsed` is negative. How many are there, and what percentage of the dataset do they represent?

2. Run `validate_domain_ranges()` with an appropriate rule for `days_elapsed`. What does the output tell you?

3. You have three options for handling these records:
   - Option A: Remove them entirely
   - Option B: Set them to 0 (treating the negative as a data entry error)
   - Option C: Flag them and keep them in the dataset with a `data_quality_flag` column

   For each option, describe: (a) what you lose analytically, (b) what you gain, and (c) when you would choose it.

4. A teammate says "just remove them, they're clearly errors." Write a two-sentence response explaining why you might not want to do that without more information.

**Requirement:** Document your final handling decision in code with a comment explaining the rationale.

## Exercise 3: Finding the Q4 Spike

You are analyzing maintenance workload patterns to forecast staffing needs for FY2025. Your manager says scheduling is roughly even month-to-month throughout the year and asks you to validate that assumption.

**Tasks:**

1. Run `check_fiscal_year_effects()` from `01_statistical_profiling.py` on the `start_date` column of the synthetic dataset. What does the output show about September volume compared to the monthly average?

2. Plot a bar chart showing monthly record counts by FY month (October through September). Color September distinctly to highlight the FY end effect.

3. If you were building a staffing forecast model and ignored this pattern, describe specifically what would go wrong with the model's predictions. What would it predict too high? What would it predict too low?

4. Propose two features you would add to a forecasting model to explicitly capture the FY seasonality pattern you observed.

**Note:** The synthetic dataset has an injected September spike. Your `check_fiscal_year_effects()` function should detect it automatically.

## Exercise 4: Procurement Threshold Analysis

The `contract_value` field in the synthetic DoN dataset represents maintenance contract values. Federal procurement has a simplified acquisition threshold at $250,000 that changes the contracting procedure above that amount.

**Tasks:**

1. Compute what percentage of contracts in the dataset fall within a 5% band just below $250,000 (i.e., between $237,500 and $250,000). Compare this to the percentage that falls in a 5% band just above $250,000 (between $250,000 and $262,500).

2. Given the synthetic dataset's log-normal distribution, what would you expect the ratio of "below $250K band" to "above $250K band" to be under random chance? Is the ratio you observed meaningfully different?

3. Write a SQL query (or pandas equivalent) that would identify all vendors who have at least five contracts in the $237,500–$250,000 range within the same fiscal year. These would be candidates for a contract-splitting review.

4. A program officer tells you that legitimate contracting decisions cause clustering below the simplified acquisition threshold because contracting officers intentionally use that vehicle. How does this change your interpretation of the clustering you found?

## Exercise 5: The Full EDA Report

This exercise puts together everything from the chapter into a complete EDA output.

**Setup:** Use the synthetic DoN maintenance dataset from `generate_synthetic_don_dataset(n_rows=50_000)`.

**Tasks:**

1. Run the full profiling sequence from `01_statistical_profiling.py`:
   - `profile_dataframe()`
   - `detect_duplicates()` with business key `["work_order_id", "hull_number", "start_date"]`
   - `validate_domain_ranges()` for `days_elapsed`, `contract_value`, `priority_code`, and `work_order_status`
   - `analyze_null_patterns()` for `completion_date` by `source_system`
   - `check_fiscal_year_effects()` on `start_date`

2. Using the outputs from Step 1, call `generate_quality_report()` to produce a structured data quality assessment.

3. The quality report will identify one or more HIGH severity findings. For each HIGH severity finding:
   - State what the finding is in plain English
   - Describe what would happen to a predictive maintenance model trained on unresolved data
   - Describe the specific remediation step (not just "fix it")

4. Write a one-page data quality memo suitable for sending to a program manager. It should contain:
   - Dataset description (rows, columns, date range, source systems)
   - The three most significant findings
   - Your recommended disposition for each finding
   - A clear statement of what the data can and cannot support until the HIGH findings are resolved

**Deliverable format:** The memo should be written in plain prose, not bullet lists. A program manager will read it and sign off. Write for that audience.

## Exercise 6: Platform Selection Scenario

You do not need to write code for this exercise. You need to think through the platform decision.

**Scenario:** You are starting a new analysis project. You have been given access to a procurement dataset on Advana. You need to understand the data well enough to propose an analytical approach to the program office in one week.

For each of the following tasks, state which platform you would use first and explain why in one sentence:

1. Understand what columns exist and what they represent, including any known quality issues.

2. Profile null rates, cardinality, and data type consistency for all 47 columns.

3. Quickly test the hypothesis that obligation amounts follow a log-normal distribution by looking at a histogram.

4. Determine whether records from DoD agency "A" have a different distribution of contract values than records from agency "B."

5. Find all records that have an obligation amount in the top 1% AND a competition type of "Sole Source" — and understand what other characteristics those records tend to have.

6. Produce a reproducible notebook that your team lead can review and that will serve as documentation for the data quality decision you made.

**Then:** Looking at your answers, is there a single platform that handles all six tasks well? If not, what does that tell you about how to structure your week?
